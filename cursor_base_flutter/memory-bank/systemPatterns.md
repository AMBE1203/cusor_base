# System Patterns

## Architecture

Flutter app at repo root; layered structure can grow with features (domain / data / presentation as needed).

## Tooling

- **FVM:** `.fvmrc` pins Flutter (e.g. 3.41.6); prefer `fvm flutter` / `fvm dart` in this repo.
- **Analysis:** `analysis_options.yaml` + `flutter_lints`.

## Conventions

- Memory Bank: `memory-bank/tasks.md` for active work; archive completed work under `memory-bank/archive/`.
